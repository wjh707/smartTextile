## 📋 岗位职能详解

### 1️⃣ SM-TC：数据采集工程师

**核心职责：**
- 定时采集五大类信源（纺织产业、学术技术、专利前沿、标准合规、市场企业）
- 原始数据按时间戳存档，不覆盖
- 数据去重、清洗、合并到主数据文件
- 监控信源可用性，发现断源及时告警

**信源覆盖范围：**

| 类别 | 关键信源 |
|------|---------|
| 纺织产业 | 中国纺织工业联合会官网、各专委会公众号、《中国纺织报》、《纺织中国在线》、《产业用纺织品》 |
| 学术 | Springer Nature / ScienceDirect 智能纺织品相关论文、预印本平台 |
| 专利 | 国知局、Patsnap 智能纺织品相关专利 |
| 标准 | 国家标准化管理委员会、全国标准信息公共服务平台 |
| 市场 | QYResearch / Fortune BI / IMARC 报告、上市公司年报 |

**产出：** `data/raw/{timestamp}/` → `data/smart_textile_data.json`

---

### 2️⃣ SM-TA：标签分析工程师

**核心职责：**
- 双轴标签体系自动标注（每条情报打上技术类型 + 应用场景 + 智能级别）
- NLP提取关键事件：技术突破、企业合作、政策要点、融资动态
- 识别交叉热点（如某个技术在多个场景的渗透率变化）
- 生成标签统计报告，反馈给看板工程师

**标签体系：**

**X轴 — 技术类型标签：**
| 代码 | 技术 |
|------|------|
| `conductive-fiber` | 导电纤维 |
| `flexible-sensor` | 柔性传感器 |
| `energy-harvesting` | 能量采集 |
| `embedded-circuit` | 嵌入式电路 |
| `communication` | 通信模块 |
| `smart-coating` | 智能涂层/整理 |
| `integration` | 纺织-电子集成工艺 |

**Y轴 — 应用场景标签：**
| 代码 | 场景 |
|------|------|
| `healthcare` | 医疗健康 |
| `sports-fitness` | 运动健身 |
| `military-defense` | 国防军事 |
| `fashion-entertainment` | 时尚娱乐 |
| `industrial-safety` | 工业安全 |
| `smart-home` | 智能家居 |
| `wearable-computing` | 可穿戴计算 |

**Z轴 — 智能化级别：**
| 级别 | 说明 |
|------|------|
| `passive` | 被动智能（仅感知，不响应） |
| `active` | 主动智能（感知+响应） |
| `ultra` | 超智能（感知+响应+自适应） |

**产出：** `data/labeled_data.json`, `data/cross_analysis.json`

---

### 3️⃣ SM-TB：前端/看板工程师

**核心职责：**
- 开发四大监测看板（技术演进雷达图、应用场景热力图、标准合规跟踪、产业竞争图谱）
- 数据驱动可视化更新（数据文件变更后看板自动生效）
- 多端适配（PC / 手机）
- 看板间联动与跳转

**四大看板技术规格：**
| 看板 | 图表类型 | 数据源 |
|------|---------|--------|
| 技术演进雷达图 | ECharts 雷达图/折线图 | `data/cross_analysis.json` |
| 应用场景热力图 | ECharts 热力图/散点图 | `data/labeled_data.json` |
| 标准合规跟踪 | 时间线/表格 | `data/standards.json` |
| 产业竞争图谱 | 力导向图/Sankey图 | `data/industry_map.json` |

**产出：** `dashboard/*.html`

---

### 4️⃣ SM-TR：报告编辑

**核心职责：**
- 按周/月/季度自动生成结构化情报摘要
- 格式化为微信/飞书可推送的简报
- 支持CEO自定义专题报告生成
- 整合三级预警事件到报告中

**报告类型：**
| 类型 | 频率 | 格式 | 推送渠道 |
|------|------|------|---------|
| 每日简报 | 每日 | 简短markdown | 微信/飞书 |
| 周报 | 每周一 | 详细markdown | 微信/飞书 |
| 月报 | 每月1日 | HTML/PDF | 飞书文档 |
| 专题报告 | 按需 | HTML | 飞书文档 |

**产出：** `data/reports/`

---

### 5️⃣ SM-TA2：高级分析师

**核心职责：**
- 深度产业链分析（各环节企业分布、竞争强度、技术依赖关系）
- 技术路线预测（基于专利/论文数据推演发展趋势）
- 预警研判（新情报是否触发红色/黄色预警，需分析师确认）
- 竞争格局报告（核心企业动态跟踪、市场份额变化）
- 跨模块交叉洞察（如专利→企业→市场趋势联动分析）

**触发条件：**
| 触发 | 行动 |
|------|------|
| 红色预警事件 | 立即出研判报告 |
| 黄色预警事件 | 24小时内出评估 |
| 月度/季度报告 | 出深度分析章节 |
| CEO直接指派 | 按需出专题报告 |

**产出：** `data/analysis/`, `data/early_warnings.json`

---

### 6️⃣ SM-TL：跨语言情报员

**核心职责：**
- 日韩英三语信源的采集与翻译
- 重点监测：日本（东丽/帝人）、韩国（首尔大学/KAIST柔性电子）、美国（MIT/Google Jacquard/杜邦）
- 关键外文情报的中文摘要整合
- 非中文专利摘要翻译

**覆盖语种：**
| 语种 | 重点关注领域 |
|------|-------------|
| 🇯🇵 日语 | 导电纤维（东丽/帝人）、碳纤维纺织品、日本产业技术综合研究所 |
| 🇰🇷 韩语 | 柔性电子织物、韩国电子通信研究院(ETRI)、首尔大学 |
| 🇬🇧 英语 | MIT Media Lab、Google Jacquard、杜邦、Hexoskin |
| 🇨🇳 中文（已有） | 中国纺织工业联合会、东华大学、香港理工大学 |

**产出：** `data/international/`

---

### 7️⃣ SM-QC：QC监工

**核心职责：**
- 每日检查各AI员工任务完成情况
- 监控数据采集覆盖率（五大信源是否都跑通）
- 检查看板可用性（页面打开、图表渲染、数据更新）
- 检查标签覆盖率（新数据是否及时标注）
- 发现进度滞后或质量问题时报警
- 生成QC日报，推送至CEO

**检查清单（每日）：**
- [ ] 数据采集：五大信源 ← 各信源是否正常
- [ ] 标签标注：新数据是否已标注
- [ ] 看板状态：四大看板是否可正常打开
- [ ] 报告生成：简报是否按时产出
- [ ] 预警处理：红色/黄色预警是否被响应
- [ ] 数据更新：stats.json 时间戳是否为今日

**产出：** QC日报推送至微信/飞书

## 🔄 协作流程

```
数据采集 (SM-TC) → 原始数据
    ↓
标签标注 (SM-TA) → 标注数据
    ↓                        ↘
看板渲染 (SM-TB)             报告生成 (SM-TR)
    ↓                        ↓
用户查看                    用户接收
    ↑                        ↑
QC监工 (SM-QC) ← 每日检查 → 预警研判 (SM-TA2)
                              ↑
                         跨语言情报 (SM-TL)
```

## 🗓️ 启动阶段计划

### Phase 1：骨架搭建（MVP）
- SM-TB（看板工程师）：用模拟数据先搭四大看板骨架
- SM-QC（QC监工）：看板稳定后立即启用

### Phase 2：数据接入
- SM-TC（数据采集）：接入第一篇真实信源
- SM-TA（标签分析）：建立双轴标签体系，手动打标验证

### Phase 3：智能增强
- SM-TL（跨语言）：接入外文信源
- SM-TA2（分析师）：预警机制上线
- SM-TR（报告编辑）：自动化报告

---

## ⚠️ 运作规则

1. **每个AI员工由CEO（用户）直接指派任务**，或由PM（Hermes）发起、CEO确认后执行
2. **QC监工自动运行**，每日输出进度报告
3. **出现红色预警** → 分析师紧急介入 → CEO决策
4. **月度回顾** → PM汇总各员工绩效 → CEO评估下一步方向
